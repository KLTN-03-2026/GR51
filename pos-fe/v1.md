# Tài Liệu Tổng Quan Hệ Thống Smart Cafe POS (v1)

Tài liệu này cung cấp một cái nhìn tổng quan về kiến trúc, cấu trúc thư mục, các module chức năng và luồng dữ liệu của hệ thống **Smart Cafe POS** (ứng dụng phía Client/Frontend).

---

## 1. Kiến Trúc Mở Rộng & Công Nghệ Cốt Lõi
Hệ thống được phát triển dựa trên mô hình **MVVM (Model - View - ViewModel)** để tách biệt rõ ràng giữa giao diện người dùng (UI) và logic nghiệp vụ.

- **Framework**: Flutter (SDK ^3.10.8)
- **Quản lý trạng thái (State Management)**: `provider` (gói Provider v6.1.5)
- **Kết nối API (Networking)**: `http` (Tương tác với Laravel Backend chuẩn RESTful API)
- **Lưu trữ cục bộ (Local Storage)**: `shared_preferences` (Lưu trữ Token đăng nhập, cấu hình thông tin phiên)

---

## 2. Cấu Trúc Thư Mục & Danh Sách Tệp (Files) Chi Tiết
Dự án được tổ chức gọn gàng trong thư mục `lib/`, dưới đây là danh sách chi tiết các file trong các thư mục cốt lõi:

### 2.1 `/models/` (Dữ liệu & DTO)
- `ban_model.dart`: Mô hình dữ liệu Bàn.
- `cart_item_model.dart`: Mô hình từng món hàng trong Giỏ hàng (Cart).
- `inventory_model.dart`: Mô hình dữ liệu của Nguyên vật liệu Tồn kho.
- `item_options_modal.dart`: Tệp chứa cấu trúc dữ liệu và modal UI phục vụ cho việc chọn option món ăn (Size, Topping...).
- `khu_vuc_model.dart`: Mô hình Khu vực để gom nhóm các Bàn.
- `menu_model.dart`: Mô hình Sản phẩm hiển thị trong Menu bán hàng.
- `order_model.dart`: Mô hình Đơn hàng chi tiết.

### 2.2 `/services/` (Giao tiếp Ngoại vi)
- `api_service.dart`: Chứa toàn bộ các hàm gọi HTTP RESTful API (hỗ trợ Bearer Token) với các endpoint từ hệ thống backend.

### 2.3 `/views/` (Giao diện Màn hình & Layouts)
Đại diện cho Tầng UI, hiển thị thông tin cho người dùng tương tác:
- `account_view.dart`: Màn hình Tài khoản & thiết lập.
- `inventory_view.dart`: Màn hình Quản lý Tồn kho.
- `kds_view.dart`: Màn hình hiển thị danh sách đơn cho KDS tổng quan (nếu dùng tách biệt).
- `main_layout.dart`: Bộ điều hướng Navigation Sidebar chính (Nền móng của toàn ứng dụng).
- `menu_view.dart`: Màn hình chức năng Bán hàng (POS).
- `order_history_view.dart`: Màn hình tra cứu Lịch sử Đơn đã chốt.

**Các thư mục phụ chứa giao diện con (`/views/`):**
- **`/auth/`** -> `login_view.dart`: Màn hình cung cấp giao diện Đăng nhập.
- **`/modals/`**:
  - `payment_modal.dart`: Modal thanh toán, xác nhận chốt bill.
  - `table_selection_modal.dart`: Modal chọn Bàn/Khu vực.
- **`/orders/`** -> `order_list_view.dart`: Màn hình KDS/Xử lý đơn pha chế.

### 2.4 `/views/viewmodels/` (Logic Nghiệp vụ & State Management)
Các Providers chứa logic nối với API và điều khiển giao diện:
- `auth_viewmodel.dart`: Quản lý phiên đăng nhập và định tuyến Auth.
- `cart_viewmodel.dart`: Quản lý các món thêm vào giỏ, tổng số tiền.
- `inventory_viewmodel.dart`: Tìm kiếm và nhận dữ liệu Tồn kho.
- `kds_viewmodel.dart`: Quản trị state liên quan tới hiển thị cho KDS.
- `menu_viewmodel.dart`: Quản lý sản phẩm load được từ danh mục Menu.
- `order_viewmodel.dart`: Handle trạng thái đơn và giao tiếp update API.
- `shift_viewmodel.dart`: Logic thao tác liên quan tới chuyển Ca/Kết ca làm việc.
- `table_selection_viewmodel.dart`: Map dữ liệu của khu vực và các Bàn trống.

## 3. Các Module & Tính Năng Trọng Tâm

Hệ thống cung cấp 5 Module lớn được hiển thị qua Sidebar bên trái trong màn hình chính:

1. **Bán Hàng (Menu/Sales)**:
   - Hiển thị danh mục sản phẩm, hỗ trợ lựa chọn thuộc tính sản phẩm (size, topping, độ ngọt, lượng đá).
   - Quản lý giỏ hàng và danh sách món hiện tại (`CartViewModel`).
   - Lựa chọn điểm dùng đồ: "Dùng tại bàn" (`TableSelectionModal`) hoặc "Mang về".
   - Gọi API thanh toán và đẩy đơn hàng thông báo cho bộ phận pha chế.

2. **Xử Lý Đơn (KDS - Kitchen Display System)**:
   - Nơi tiếp nhận thông tin đơn hàng mới dành cho pha chế/nhà bếp theo thời gian thực (được tải lên từ DB).
   - Cập nhật trạng thái từng đơn (Từ '*Chờ pha chế*' chuyển qua '*Đã pha chế*'). Đặc biệt, khi cập nhật trang thái, back-end sẽ tự động trừ lượng tồn kho tương ứng.
   - Tích hợp tính năng xem chi tiết 'Công thức' ngay tại đơn.

3. **Lịch Sử Đơn (Order History)**:
   - Tra cứu lại các đơn hàng đã tạo trong ca làm việc.
   - Thống kê thông tin tổng thành tiền và phương thức thanh toán.

4. **Quản Lý Tồn Kho (Inventory Management)**:
   - Hiển thị tình trạng nguyên vật liệu trực quan (`InventoryView`).
   - Phân mức cảnh báo thông minh:
     - 🔴 **Đỏ**: Hết hàng (Tồn kho ≤ 0).
     - 🟠 **Cam**: Sắp hết (Tồn kho ≤ Mức cảnh báo).
     - 🟢 **Xanh**: Mức an toàn.
   - Tích hợp tìm kiếm nhanh nguyên liệu, nút tải lại (Refresh) đồng bộ liên tục với DB.

5. **Tài Khoản & Ca Làm Việc (Account & Shift)**:
   - Thông tin tài khoản nhân viên. Thực hiện các thao tác mở ca và kết ca.
   - Chốt số dư tiền mặt/chuyển khoản để khớp với số liệu trên hệ thống lúc giao ca (`ShiftViewModel`).

## 4. Luồng Dữ Liệu (Data Flow) & Bảo Mật

- **Xử lý Request & Bất đồng bộ**: Mọi tác vụ trao đổi với Backend (localhost:8000/api/v1) đều qua `ApiService`. Các luồng gọi API đã bọc `try-catch`, kiểm soát theo HTTP code, và handle biệt lệ JSON rất tốt.
- **State Management**: ViewModel tải dữ liệu qua Service, cập nhật các List/Obj sau đó gọi `notifyListeners()`. Ở UI, dùng `context.watch`, `context.read`, hoặc `Consumer` để lắng nghe sự thay đổi và vẽ lại duy nhất những điểm cần thiết.
- **Bảo mật**: Các Route được nhúng Token xác thực `Authorization: Bearer <Token>` (lấy từ log in qua shared prefs). Nếu Server trả về lỗi 401 thì hệ thống quăng ra `UnauthorizedException` chuẩn hóa.

---

## 5. Tổng Kết Đánh Giá

- **Cấu trúc chuẩn, Clean architecture**: Rất rõ ràng, tuân thủ đúng định hướng MVVM của nền tảng Flutter. Logic không bị "dính" vào Widget.
- **Sẵn sàng Scalable (Mở rộng)**: Việc chia file, ViewModel gọn gàng giúp quy trình bảo trì, test hoặc phát triển module mới (vd khách hàng thành viên, voucher) sẽ rất tách biệt và không ảnh hưởng code lỗi tới những module cũ.
- **Giao diện thân thiện UX/UI**: Flow đặt hàng, pha chế nhanh và liên thông với nhau, sử dụng chung mã màu (vàng cam/dark xám) nên mang lại trải nghiệm chuyên nghiệp cho quán Cafe.
